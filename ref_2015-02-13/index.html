Abstract

Main memory database systems (MMDBs) are viable solutions for many applications. Different application domains require different kinds of indexing and there is a large of number main memory indexing implementations developed. However, adding an index structure into a database system can be challenging. Mexima (Main-memory External Index Manager) provides an MMDB where new main memory index structures can be plugged in without changing the index implementations. To transparently utilize new indexes in queries, Mexima provides index property tables populated by the extension developer. They enable automatic transformation of query fragments to call index-supported operations. When indexed attributes are hidden inside complex numerical expressions, an algebraic query transformation algorithm reasons on the expressions to expose hidden indexes. Furthermore, the index property tables are also used for validating the correctness of an index implementation by executing automatically generated test queries. Experiments show that the overhead of plugged in indexes is low compared to the stand-alone C/C++ implementations. Substantial performance gains are shown by the index exposing rewrite mechanisms.
